### ⚡ MCP Server Superpowers: The Three Primitives

An MCP server gives an AI three ways to interact with your world. Think of them as the AI's **Eyes**, **Brain**, and **Hands**.

| Power | Analogy | What it is | Real-World Case |
| :--- | :--- | :--- | :--- |
| **1. Resources** | **The Eyes** | Read-only data. | "Read this `log.txt` file." |
| **2. Prompts** | **The Brain** | Pre-set instructions. | "Use the 'Code Review' template." |
| **3. Tools** | **The Hands** | Executable actions. | "Delete this file" or "Send a Slack." |

---

### 🛠️ Deep Dive: Tools (The "Action" Layer)

**Tools** are the most powerful part of MCP. They turn a "Chatbot" into an **AI Agent**. Instead of just talking about a problem, the AI can actually use a tool to solve it.

#### What makes a Tool?
* **JSON Schema:** A clear "instruction manual" that tells the AI exactly what inputs it needs (e.g., "I need a `filename`").
* **Single Focus:** A tool should do one thing well (e.g., `write_file`).
* **Safety Switch:** The Client (you) must click **[Approve]** before any action actually happens.

#### How the AI Uses Tools (The Flow)
1.  **Discovery:** The AI asks, "What tools do I have?" and gets a list like `[read_db, send_email]`.
2.  **Decision:** Based on your request ("Fix the bug and email the team"), the AI picks the best tool for the job.
3.  **Invocation:** The AI sends a `tools/call` message. The server runs the code and sends back the result.

### ⏱️ Async vs. Synchronous Tools

If you're using **FastMCP**, it handles both types of functions seamlessly:

* **Async Tools (`async def`):** The "Gold Standard." Use these for anything that takes time, like calling an API or reading a large database. It keeps the server "awake" and responsive.
* **Sync Tools (`def`):** Best for quick, local math or simple logic.

---

### 📢 Notifications (Stay Updated)

You don't want the AI trying to use a tool that doesn't exist anymore. FastMCP uses **`tools/list_changed`** notifications to keep everyone in sync.

* **Automatic Updates:** When you add, remove, or disable a tool while the server is running, it "shouts" a notification to the client.
* **No Manual Refresh:** The client (like Claude Desktop) sees the notification and automatically updates its list of "superpowers" without you needing to restart anything.
* **Initialization Rule:** Notifications aren't sent during the very first startup (initialization)—only when things change *after* the connection is already live.

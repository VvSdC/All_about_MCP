### ⚡ MCP Server Superpowers: The Three Primitives

An MCP server gives an AI three ways to interact with your world. Think of them as the AI's **Eyes**, **Brain**, and **Hands**.

| Power | Analogy | What it is | Real-World Case |
| :--- | :--- | :--- | :--- |
| **1. Resources** | **The Eyes** | Read-only data. | "Read this `log.txt` file." |
| **2. Prompts** | **The Brain** | Pre-set instructions. | "Use the 'Code Review' template." |
| **3. Tools** | **The Hands** | Executable actions. | "Delete this file" or "Send a Slack." |

---

### 🛠️ Deep Dive: Tools (The "Action" Layer)

**Tools** are the most powerful part of MCP. They turn a "Chatbot" into an **AI Agent**. Instead of just talking about a problem, the AI can actually use a tool to solve it.

#### What makes a Tool?
* **JSON Schema:** A clear "instruction manual" that tells the AI exactly what inputs it needs (e.g., "I need a `filename` and a `text_content`").
* **Single Focus:** A tool should do one thing well—like `write_file`—rather than trying to manage an entire project.
* **Safety Switch:** By default, the AI can't just run a tool. The Client (you) must click **[Approve]** before the action happens.

#### How the AI Uses Tools (The Flow)
1.  **Discovery:** The AI asks, "What tools do I have?" and gets a list like `[read_db, write_db, send_email]`.
2.  **Decision:** Based on your request ("Fix the bug and email the team"), the AI decides it needs to use `read_db` and `send_email`.
3.  **Invocation:** The AI sends a `tools/call` message. The server executes the code and sends back the result.

#### Common Real-World Examples:
* **File System:** Creating, moving, or deleting files on your hard drive.
* **Web Access:** Fetching live data from a website or searching Google.
* **Databases:** Running a SQL query to find a specific user.
* **APIs:** Posting a message to a Discord channel or triggering a GitHub Action.

> **Key takeaway:** While **Resources** are for reading and **Prompts** are for guiding, **Tools** are for **doing**. They are model-controlled, meaning the AI decides *when* to pick up the tool to finish the job you gave it.

### MCP server provides three types of capabilities:

1. Resources: Context and data, for the user or the AI model to use. Structured data like files, database queries, or API responses, providing information to the LLM.

2. Prompts: Templated messages and workflows for the users. Predefined instructions or Pre-written templates that influence the LLM's response generation i.e that helps users accomplish tasks.

3. Tools: Executable functions that allow the LLM to interact with external services, such as triggering an API call or updating records. Functions that can be called by LLM (with user approval)



#### Tools

Tools are powerful primitive in the Model Context Protocol (MCP) that enable MCP servers to expose executable functions that can be discovered and invoked (executed at the server-side) by the MCP clients and used by LLMs of the AI-Agent (MCP agents) to perform actions. Through tools, LLMs can interact with external systems, perform computations, and take actions in the real world. The MCP client provides methods to discover available tools and execute them with arguments. Tools are well defined functions with:
- JSON schema definitions (that specify their inputs and outputs)
- Single, focused operations (that perform one specific task)
- User approval requirement (before execution for security)

Common examples are:
- File operations (read and write)
- Web browsing and data fetching
- Database queries
- API calls to external services
- System commands.

Tools are designed to be model-controlled, that means tools are exposed from MCP servers to MCP clients with the intention of the AI model being able to automatically invoke them. Key aspects of tools are:

- Discovery: Clients obtain a list of available tools by sending a tools/list request
- Invocation: Tools are called using the tools/call request, where servers perform the requested operation and return results.
- Flexibility: Tools can range from simple calculations to complex API interactions.

Tools are identified by unique names and can include descriptions to guide their usage. However, the tools represent dynamic operations that can modify state or interact with external systems. Clients can list available tools at any time and tools can be added or removed during run time.

# Model Context Protocol (MCP) Mastery

Welcome to the definitive guide for the **Model Context Protocol (MCP)**. Think of MCP as a **USB-C port for AI**. Just as a universal port allows your laptop to connect to any monitor or hard drive, MCP allows any AI (like Claude or Gemini) to connect to your databases, local files, or APIs using a single, secure standard.

## 🏗️ Architecture & Ecosystem

MCP operates on a simple **Client-Server model**. 

| Component | Role | Real-World Example |
| :--- | :--- | :--- |
| **MCP Host** | The app you use. | Claude Desktop, VS Code. |
| **MCP Client** | The internal bridge. | The "translator" inside the app. |
| **MCP Server** | The data provider. | A script that reads your Google Calendar. |

## 🚚 Transport Layer: How Data Moves

The transport layer is the "physical pipe" that carries messages. MCP is "transport-agnostic," meaning it doesn't care if the data travels through a local cable or the internet.

### 1. STDIO Transport (Local)
This is for apps running on the **same computer**. The client starts the server as a background process.
* **How it works:** They talk via `stdin` and `stdout` (standard input/output).
* **Logging:** To keep the data clean, errors and logs are sent through `stderr`.

`# Example: Starting a local python MCP server`
`python my_server.py`

### 2. Streamable HTTP (Remote)
Used for **web-based** tools. It uses HTTP POST for sending messages and **Server-Sent Events (SSE)** for streaming data back.

`# Sending a request via HTTP POST`
`POST /mcp/message`
`Content-Type: application/json`
`{ "jsonrpc": "2.0", "method": "tools/call", "params": {...}, "id": 1 }`

## 🛠️ Protocol Capabilities

### What Servers give to AI:
* **Resources:** (Read-only) "Here is the code from `main.py`."
* **Prompts:** (Instructions) "Help the user write a SQL query for this DB."
* **Tools:** (Actions) "Create a new folder named 'Backup'."

`# A simple Tool snippet in Python`
`@mcp.tool()`
`def create_folder(name: str):`
`    os.makedirs(name)`
`    return f"Folder {name} created!"`

### What Clients give to Servers:
* **Sampling:** The server can ask the AI, "Hey, I found an error in this file, can you fix the syntax for me?"
* **Roots:** The client tells the server, "You are only allowed to see files in `/User/Projects`."

## 🛡️ Security & Session Management

To keep things secure, MCP uses **Stateful Sessions**.

1. **Handshake:** The server gives the client a unique ID.
2. **Identity:** Every message must include that ID: `MCP-Session-ID: "sess_12345"`.
3. **Safety:** Local servers are forced to stay on `127.0.0.1` so outsiders can't hack in.

## 📡 JSON-RPC: The Heartbeat

JSON-RPC is a simple text message format. A typical request looks like this:

`{`
`  "jsonrpc": "2.0",`
`  "method": "resources/read",`
`  "params": { "uri": "file:///logs/app.log" },`
`  "id": "req_001"`
`}`

The server then replies with a result or an error if the file wasn't found.

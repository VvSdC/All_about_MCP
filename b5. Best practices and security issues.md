# 🛡️ MCP Security: Building a "Zero Trust" Server

Building an MCP server means giving an AI "hands" to touch your data. To keep those hands from doing damage, you need a **Zero Trust** mindset. Here is how to build a server that is both professional and "un-hackable."

---

## 🛠️ Best Practices: The Professional Standard

### 1. Choosing the Right Pipe (Transport)
* **Local (STDIO):** The "Desktop Standard." Fast and private for local apps like VS Code.
* **Remote (HTTP/SSE):** The "Cloud Standard." Requires **API Keys** or **OAuth2** to make sure only authorized users can connect.

### 2. Don't Trust the LLM (Input Validation)
AI models can "hallucinate" weird inputs. Always use a validator like **Pydantic** to double-check everything the AI sends.
* **Example:** If a tool asks for a `limit`, make sure it’s a number and not a string like `"DROP TABLE users"`.

### 3. User Experience (Progress & Timeouts)
* **Progress:** For long tasks (like scanning 1,000 files), send "Progress Updates" so the user knows the AI isn't stuck.
* **Timeouts:** Kill any task that takes longer than 30 seconds. This prevents a single "frozen" tool from crashing your server.

---

## 🔐 Security: Guarding the Gates

### 1. The "Confused Deputy" (The Proxy Trap)
This happens when a middle-man server is tricked into using its high-level permissions to do something bad for a sneaky user.
* **The Fix:** Never use one master ID. Every new connection must get **explicit permission** from the user before it can access sensitive data.

### 2. The Token Taboo
**Rule:** Never play "Pass the Token."
* **The Risk:** If an AI client gives you a stolen password/token and you just pass it on to GitHub, you've just helped a hacker.
* **The Fix:** Only accept tokens made specifically for **your** server. Check them yourself before doing any work.

### 3. Guarding the Session
Attackers might try to guess a `Session-ID` to "hijack" a conversation.
* **The Fix:** Use long, random, "secret" session IDs.
* **Expiration:** Make IDs expire quickly so an old ID can't be used again.
* **Sanitize:** Treat every word from the AI as "dirty" data—never run it directly as code.

---

## 📝 Security Checklist

| Category | Must-Do Action | Why? |
| :--- | :--- | :--- |
| **Connection** | Use **HTTPS** for web servers. | Stops people from "listening in" on the data. |
| **Local Servers** | Use `127.0.0.1` (Localhost). | Prevents other people on your Wi-Fi from connecting. |
| **Errors** | Hide internal paths/code. | Don't give hackers a "map" of your computer. |
| **Rate Limits** | Set a cap on request size. | Prevents "Spam attacks" that could freeze your PC. |

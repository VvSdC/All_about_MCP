## 📡 The Four Ways to Talk (Message Types)

The Model Context Protocol (MCP) uses **JSON-RPC** as its "envelope." Think of these four message types as the different ways people communicate in a restaurant.

---

### 1. Requests (The Order)
A **Request** is sent when one part of the system wants the other to **do something** or **provide data**. It *always* expects an answer.

'# Asking for a specific tool to run'
'{'
'  "jsonrpc": "2.0",'
'  "method": "tools/call",'
'  "params": { "name": "get_weather", "arguments": {"city": "Delhi"} },'
'  "id": 1'
'}'

* **Real-world:** You ask a waiter, "Can I have the menu?" You are the requester, and you won't leave until you get that menu or an answer.

---

### 2. Results (The Delivery)
A **Result** is the "Mission Accomplished" message. It uses the same `id` as the request so the AI knows exactly which question is being answered.

'# The answer to the weather request'
'{'
'  "jsonrpc": "2.0",'
'  "result": { "content": [{"type": "text", "text": "It is 25°C and sunny."}] },'
'  "id": 1'
'}'

* **Real-world:** The waiter brings the menu to your table.

---

### 3. Errors (The "Out of Stock")
If a request fails (e.g., the AI asks for a file that doesn't exist), the system sends an **Error** instead of a Result.

'# Telling the AI the tool doesn't exist'
'{'
'  "jsonrpc": "2.0",'
'  "error": { "code": -32601, "message": "Tool not found" },'
'  "id": 1'
'}'

* **Real-world:** The waiter says, "Sorry, we don't serve breakfast after 11 AM."

---

### 4. Notifications (The Announcement)
**Notifications** are one-way shouts. They **do not** expect a response. They are used for updates like "I'm starting up now" or "I've finished loading."

'# Telling the system the connection is ready'
'{'
'  "jsonrpc": "2.0",'
'  "method": "notifications/initialized"'
'}'

* **Real-world:** A sign on the door says, "We are now open." The sign doesn't wait for you to say "Okay" back.

---

## 📝 Comparison Table

| Message Type | Needs an Answer? | Core Parts | Real-World "Vibe" |
| :--- | :--- | :--- | :--- |
| **Request** | ✅ Yes | `method`, `id` | "Can you do this for me?" |
| **Result** | ❌ No | `result`, `id` | "Here is what you asked for." |
| **Error** | ❌ No | `code`, `id` | "I can't do that, and here's why." |
| **Notification** | ❌ No | `method` | "Just letting you know..." |
